## Fix shape

The bug was a textbook React stale-state issue. `DocPage` is the same component mounted for `/doc/A` and `/doc/B` — React reuses the instance, so `useState("")` carries `"A"` into the `B` render. The original code papered over it with a setState-in-render guard (`if (title === "" && meta.title) setTitle(meta.title)`) that only fires once.

Fix: extracted a `DocTitle({ id, initialTitle })` sub-component, mounted with `key={id}`. React unmounts and remounts on docId change, so `useState(initialTitle)` initializes fresh every time. The rename mutation moves into the sub-component alongside the state it manages.

```tsx
function DocTitle({ id, initialTitle }: { id: string; initialTitle: string }) {
  const [title, setTitle] = useState(initialTitle);
  // ...mutation + input...
}

// in DocPage:
<DocTitle key={id} id={id} initialTitle={meta.title} />
```

Both e2e specs that worked around this via `page.evaluate(fetch /api/docs)` now use the natural UI flow (`new-doc` button → fill title → blur → search). 21/21 pass.

## What was non-obvious

**setState-in-render is a smell.** The original guard violated React's "no side effects in render" rule. ESLint's `react-hooks/set-state-in-effect` doesn't catch it because it's in the render body, not an effect. Code review would have flagged it; the bug went unnoticed because the in-flight `setState` happened to work for the first hydration. Lesson: prefer `key`-based remounts for "reset state when prop X changes" over either setState-in-render or extra effects.

**`key` propagates remount through Suspense and lazy children.** Worried briefly that `key` would conflict with React.lazy boundary or break the editor; it doesn't — only the keyed sub-component remounts. Editor + provider stay scoped to their own `KnotEditor` mount.
