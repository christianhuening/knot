# Tables Implementation Plan (Plan 29)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** WYSIWYG tables in the editor with full GFM markdown round-trip. Headers, alignment per column, add/remove rows/cols, copy/paste between cells.

**Architecture:**
- Tiptap extensions: `@tiptap/extension-table`, `-table-row`, `-table-cell`, `-table-header`.
- Schema nodes: `table`, `table_row`, `table_cell`, `table_header`. Cells contain `block+` (so you can put paragraphs, lists, etc. inside a cell — matches Notion/Outline).
- Markdown: GFM tables. pulldown_cmark already supports parsing them via `Options::ENABLE_TABLES` (used in `public.rs`, need to mirror in `knot-markdown::from_markdown`).
- Serializer emits one row per line with `|` separators and the alignment row for headers (`|---|:---:|---:|`).

---

## Tasks

### T1: Schema — table nodes
- `tools/schema.json`: add `table` (block, content `table_row+`), `table_row` (content `(table_cell | table_header)+`), `table_cell` (block, content `block+`, attrs `colspan` (default 1), `rowspan` (default 1), `colwidth` (nullable array of int)), `table_header` (same as cell but distinguished for serializer).
- Per-column alignment lives on `table` as `align: ("left"|"center"|"right"|null)[]`.
- Regen.

### T2: knot-markdown — parse GFM tables
- `from_markdown.rs`: enable `Options::ENABLE_TABLES`. Walk pulldown's `Tag::Table` / `TableHead` / `TableRow` / `TableCell` events; build a `table` node with the alignment array from the parser's `Vec<Alignment>`.

### T3: knot-markdown — emit GFM tables
- `to_markdown.rs`: on `table`, emit:
  - First row from `table_row` containing `table_header` cells → `| h1 | h2 | h3 |`.
  - Alignment row from `table.attrs.align` → `|---|:---:|---:|`.
  - Body rows from subsequent `table_row` containing `table_cell` cells.
- Cell content is serialized as the inline content of its first paragraph; nested blocks degrade gracefully (paragraphs separated by `<br>` or `\n`, per GFM conventions).
- Round-trip fixture `tables.md`.

### T4: Tiptap config
- `web/src/features/editor/extensions.ts`: add the four table extensions with the schema-aligned names.
- Cell selection behavior, tab key navigation: rely on extension defaults; tune later if jarring.

### T5: Toolbar UI
- New "Insert table" button (icon `Table` from Lucide) opens a small popover with a 5×5 grid picker (Notion-style).
- Once a table is inserted, an inline toolbar appears above the focused cell with: add row above/below, add col left/right, delete row/col, set alignment, toggle header row.
- testIds `toolbar-table`, `table-add-row`, etc.

### T6: e2e — `tables.spec.ts`
- Insert 3×3, type in cells, refresh, content persists.
- Export markdown, assert GFM table syntax.
- Two-user converge: both edit different cells, both see both edits.
