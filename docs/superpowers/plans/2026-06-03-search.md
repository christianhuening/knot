# Full-Text Search Implementation Plan (Plan 14)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Search across every doc in a workspace by title and body content, ACL-respecting, results in the command palette. Postgres FTS is the engine — no new search service, no external index, no embeddings.

**Architecture:**
- **Index source:** `doc_markdown_cache.markdown_text` (Plan 5) for body; `documents.title` for title. Body cache is lazily filled on export and invalidated when the writer task advances `rendered_at_seq` — search results may lag editor state by a few seconds for actively edited docs. That's acceptable for v0.1.
- **Index:** add a `STORED GENERATED` `tsvector` column on `doc_markdown_cache` (English config) + GIN index. `documents.title` gets a separate `tsvector` generated column with a smaller GIN to support title prefix/word matches.
- **Query:** `SELECT documents.id, title, snippet, rank FROM documents LEFT JOIN doc_markdown_cache ON doc_id WHERE (title_tsv @@ plainto_tsquery OR body_tsv @@ plainto_tsquery) AND workspace_id = $1 AND archived = false ORDER BY rank DESC LIMIT 20`. `ts_rank_cd` for scoring, `ts_headline` for snippets.
- **ACL:** the workspace_id scope handles the workspace boundary. For each candidate doc, re-check `effective_role(workspace, doc, user)` — workspace member with ≥viewer wins. Filtering happens server-side after the FTS query.
- **Frontend:** the existing command palette already filters local doc titles. When the query is ≥3 chars, fire a debounced (200 ms) search request and merge server results with the static nav/action items. Show snippets under the title.

**Tech Stack:** Postgres FTS (`tsvector`, `plainto_tsquery`, `ts_rank_cd`, `ts_headline`). No new Rust or TS deps.

**Predecessor:** Plan 13.5 (runtime S3, HEAD `230e797`).

**Out of scope:**
- **Vector / semantic search.** Postgres FTS is enough for v0.1; pgvector is a future plan if relevance becomes a problem.
- **Cross-workspace search.** Single workspace per knot install for v0.1.
- **Faceted search** (filter by author / date / tag).
- **Search-as-you-type prefix matching.** plainto_tsquery is whole-word; prefix matching needs `to_tsquery` with `:*` suffix. Defer.
- **Stop-word customisation.** `english` dictionary is fine for v0.1.
- **Highlight markup beyond `<b>...</b>`.** ts_headline default suffices.
- **Saved searches.**

## Conventions

- **Search config:** `english`. Hard-coded for v0.1; if multi-language docs become a thing, parametrize per-workspace later.
- **Result shape:** `{ doc_id, title, snippet, rank, parent_id }`. Snippet is `ts_headline` output or, if title-only match, an empty string.
- **Snippets:** body-cache match — `ts_headline('english', markdown_text, plainto_tsquery('english', $q), 'MaxFragments=2,MinWords=5,MaxWords=15')`. Title-only matches → empty.
- **Debounce:** 200 ms in the palette. No request fires for `q.length < 3`.
- **ACL:** workspace boundary in SQL (`workspace_id = $1`); per-doc `effective_role` check in the handler. Discard docs the user can't see.
- **Limit:** server caps at 20 results; client doesn't send a limit. Workspace-wide rate limit not needed — the existing throttle bucket covers HTTP traffic.

```sql
-- Created 2026-06-03

-- doc_markdown_cache.body_tsv — full-text index of the cached markdown.
-- The markdown cache lags live editor state until the next snapshot, so
-- search is eventually consistent with edits. Acceptable for v0.1.
ALTER TABLE doc_markdown_cache
  ADD COLUMN body_tsv tsvector
  GENERATED ALWAYS AS (to_tsvector('english', coalesce(markdown_text, ''))) STORED;
CREATE INDEX doc_markdown_cache_body_tsv_idx
  ON doc_markdown_cache USING GIN (body_tsv);

-- documents.title_tsv — title-only index. Smaller and faster than body,
-- and we always have a title even before the body cache is populated.
ALTER TABLE documents
  ADD COLUMN title_tsv tsvector
  GENERATED ALWAYS AS (to_tsvector('english', coalesce(title, ''))) STORED;
CREATE INDEX documents_title_tsv_idx
  ON documents USING GIN (title_tsv);
```

```rust
//! Postgres-backed full-text search across docs in a workspace.

use async_trait::async_trait;
use sqlx::PgPool;
use thiserror::Error;
use uuid::Uuid;

#[derive(Debug, Error)]
pub enum SearchStoreError {
    #[error("sqlx: {0}")]
    Sqlx(#[from] sqlx::Error),
}

#[derive(Debug, Clone, serde::Serialize)]
pub struct SearchHit {
    pub doc_id: Uuid,
    pub parent_id: Option<Uuid>,
    pub title: String,
    pub snippet: String,
    pub rank: f32,
}

#[async_trait]
pub trait SearchStore: Send + Sync {
    async fn search(
        &self,
        workspace_id: Uuid,
        q: &str,
        limit: i64,
    ) -> Result<Vec<SearchHit>, SearchStoreError>;
}

pub struct PgSearchStore {
    pool: PgPool,
}

impl PgSearchStore {
    pub fn new(pool: PgPool) -> Self { Self { pool } }
}

#[async_trait]
impl SearchStore for PgSearchStore {
    async fn search(
        &self,
        workspace_id: Uuid,
        q: &str,
        limit: i64,
    ) -> Result<Vec<SearchHit>, SearchStoreError> {
        // ts_rank_cd is the cover-density rank; gives reasonable scores
        // across mixed title/body matches.
        let rows: Vec<(Uuid, Option<Uuid>, String, Option<String>, f32)> = sqlx::query_as(
            r#"
            SELECT d.id,
                   d.parent_id,
                   d.title,
                   CASE
                     WHEN c.body_tsv @@ plainto_tsquery('english', $2) THEN
                       ts_headline('english', c.markdown_text,
                                   plainto_tsquery('english', $2),
                                   'MaxFragments=2,MinWords=5,MaxWords=15,StartSel=<b>,StopSel=</b>')
                     ELSE NULL
                   END AS snippet,
                   GREATEST(
                     COALESCE(ts_rank_cd(d.title_tsv, plainto_tsquery('english', $2)), 0.0) * 2.0,
                     COALESCE(ts_rank_cd(c.body_tsv,  plainto_tsquery('english', $2)), 0.0)
                   )::real AS rank
              FROM documents d
              LEFT JOIN doc_markdown_cache c ON c.doc_id = d.id
             WHERE d.workspace_id = $1
               AND d.archived = false
               AND (
                     d.title_tsv @@ plainto_tsquery('english', $2)
                  OR c.body_tsv  @@ plainto_tsquery('english', $2)
                   )
             ORDER BY rank DESC
             LIMIT $3
            "#,
        )
        .bind(workspace_id)
        .bind(q)
        .bind(limit)
        .fetch_all(&self.pool)
        .await?;

        Ok(rows.into_iter().map(|(doc_id, parent_id, title, snippet, rank)| SearchHit {
            doc_id,
            parent_id,
            title,
            snippet: snippet.unwrap_or_default(),
            rank,
        }).collect())
    }
}
```

```rust
//! GET /api/search?q=<query>&limit=<n>
//!
//! Searches docs in the user's workspace by title + body, ACL-filtered.

use std::collections::HashSet;

use axum::{
    extract::{Query, Request, State},
    http::StatusCode,
    response::{IntoResponse, Response},
    routing::get,
    Json, Router,
};

use crate::auth::AuthContext;
use crate::http_error::json_err;
use crate::AppState;

#[derive(serde::Deserialize)]
struct SearchParams {
    q: String,
    #[serde(default)]
    limit: Option<i64>,
}

#[derive(serde::Serialize)]
struct SearchResponse {
    results: Vec<knot_storage::SearchHit>,
}

const MAX_LIMIT: i64 = 20;
const MIN_QUERY_LEN: usize = 2;

pub fn router() -> Router<AppState> {
    Router::new().route("/api/search", get(search))
}

async fn search(
    State(state): State<AppState>,
    Query(params): Query<SearchParams>,
    req: Request,
) -> Response {
    let Some(ctx) = req.extensions().get::<AuthContext>().cloned() else {
        return json_err(StatusCode::UNAUTHORIZED, "auth.session_required", "");
    };
    if params.q.trim().chars().count() < MIN_QUERY_LEN {
        return Json(SearchResponse { results: vec![] }).into_response();
    }
    let limit = params.limit.unwrap_or(MAX_LIMIT).clamp(1, MAX_LIMIT);
    let Some(search) = state.search.clone() else {
        return json_err(StatusCode::INTERNAL_SERVER_ERROR, "internal", "");
    };
    let Some(acl) = state.acl.clone() else {
        return json_err(StatusCode::INTERNAL_SERVER_ERROR, "internal", "");
    };

    let raw = match search.search(ctx.workspace_id, params.q.trim(), limit * 2).await {
        Ok(r) => r,
        Err(_) => return json_err(StatusCode::INTERNAL_SERVER_ERROR, "internal", ""),
    };

    let mut allowed = Vec::with_capacity(raw.len());
    let mut seen = HashSet::new();
    for hit in raw {
        if !seen.insert(hit.doc_id) { continue; }
        match acl.effective_role(ctx.workspace_id, hit.doc_id, ctx.user_id).await {
            Ok(Some(_)) => allowed.push(hit),
            _ => continue,
        }
        if allowed.len() >= limit as usize { break; }
    }
    Json(SearchResponse { results: allowed }).into_response()
}
```

```ts
import { apiFetch, type ApiResult } from "./api";

export type SearchHit = {
  doc_id: string;
  parent_id: string | null;
  title: string;
  snippet: string;
  rank: number;
};

type SearchResponse = { results: SearchHit[] };

export const searchApi = {
  async query(q: string, limit = 20): Promise<ApiResult<SearchHit[]>> {
    const params = new URLSearchParams({ q, limit: String(limit) });
    const r = await apiFetch<SearchResponse>(`/api/search?${params.toString()}`);
    if ("error" in r) return r;
    return { ok: r.ok.results };
  },
};
```

## Task 6: Command palette server search

**Files:**
- Modify: `web/src/components/CommandPalette.tsx`

- [ ] **Step 1: Integrate**

The palette currently filters a static list of doc titles + nav/actions by substring. After this task:

- For `q.length < 2`, behave as today (nav/actions only — no per-doc filtering).
- For `q.length >= 2`, fire `searchApi.query(q)` debounced 200 ms, merge server results AT THE TOP of the list with snippets shown below the title.
- Cache results in component state; cancel in-flight fetches when the query changes (use an `AbortController`).

Sketch:

```tsx
const [serverHits, setServerHits] = useState<SearchHit[]>([]);
useEffect(() => {
  if (!open || q.trim().length < 2) { setServerHits([]); return; }
  const ac = new AbortController();
  const t = setTimeout(async () => {
    const r = await searchApi.query(q.trim());
    if (ac.signal.aborted) return;
    if ("ok" in r) setServerHits(r.ok);
  }, 200);
  return () => { clearTimeout(t); ac.abort(); };
}, [q, open]);
```

Merge into `actions`:

```tsx
const serverActions: Action[] = serverHits.map((h) => ({
  id: `search:${h.doc_id}`,
  label: h.title,
  kind: "doc",
  snippet: h.snippet,
  run: () => { close(); void nav(`/doc/${h.doc_id}`); },
}));
return [...serverActions, ...staticNavActions];
```

Render snippet as a dimmed line under the label when present, using `dangerouslySetInnerHTML` with the `<b>` tags from `ts_headline`. **Sanitize first** — ts_headline only outputs `<b>` and the user-controlled query is whitespace-stripped server-side, but be defensive: build a tiny `safeHeadline(s)` that allows only `<b>…</b>`.
