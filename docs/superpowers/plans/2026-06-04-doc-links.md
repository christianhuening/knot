# Doc Links Implementation Plan (Plan 26)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task.

**Goal:** The Link toolbar/dialog gains a "Document" mode. Internal links store `knot://doc/<uuid>`, render with the target doc's live title, navigate via react-router (no full page load), and round-trip through markdown.

**Architecture:**
- The existing Tiptap `Link` mark stays as-is; we only change rendering and the dialog UI. The href just happens to use a `knot://doc/<uuid>` sentinel.
- A small `InternalLink` ProseMirror plugin (or NodeView for the mark) walks decorations and replaces the link's child text content with the live title from `docsApi.get(<uuid>)` cached via react-query. Clicks intercepted to navigate via `useNavigate`.
- Public share renderer rewrites `knot://doc/<uuid>.md` URLs to either `/p/<token>` (when the target is also shared) or strips the link to plain text (when it isn't).
- Markdown: `[<resolved title or fallback>](knot://doc/<uuid>)`. Round-trip preserves the sentinel.

**Tech stack:** existing `@tiptap/extension-link`, `react-router-dom`, react-query, knot-markdown.

## Tasks

### T1: Schema — confirm Link mark works for internal hrefs
- No schema change. `tools/schema.json` already has the `link` mark with `href` attr.
- Verify by adding a unit test that round-trips `[Doc Foo](knot://doc/<uuid>)`.

### T2: `knot_markdown` — sentinel constants
- Add `DOC_URL_PREFIX = "knot://doc/"` to `crates/knot-markdown/src/lib.rs`.
- No special parsing logic needed: `[text](url)` already round-trips as link.

### T3: Frontend — Link dialog gets a "Document" tab
- Modify `web/src/features/editor/EditorToolbar.tsx` link popover. Two segmented controls: **URL** | **Document**.
- **Document** mode: autocomplete input querying `docsApi.list({ q })` (debounce 200ms). On select, write `knot://doc/<uuid>` to the href.
- Existing autolink/manual-URL path unchanged.

### T4: Internal-link renderer
- Create `web/src/features/editor/InternalLinkExtension.ts` — a ProseMirror plugin that:
  - Finds every link mark whose href starts with `knot://doc/`.
  - Replaces the link's DOM with a `<Link to={`/doc/<id>`}>` (via React portal) showing the live title from `docsApi.get(<id>)` (cached).
  - Tooltip on hover shows the full doc breadcrumb.
- Mount in `createExtensions`.

### T5: Markdown to/from
- `to_markdown`: when emitting a link with href starting `knot://doc/`, prefer the rendered text (already in the doc) — fall back to the doc title fetched via storage if text is empty.
- `from_markdown`: no change; `[text](knot://doc/<uuid>)` already becomes a link mark.
- Round-trip fixture + tests.

### T6: Public share rewrite
- In `crates/knot-server/src/routes/public.rs`, extend `render_markdown` to also rewrite `knot://doc/<uuid>` URLs.
- For each, check whether the target doc has an active share token. If yes, emit the absolute `/p/<other_token>` URL. If no, strip the link wrapping (keep the text).

### T7: e2e
- `e2e/flows/doc-links.spec.ts`: insert internal link, navigate, rename target doc, verify the link title updates.
