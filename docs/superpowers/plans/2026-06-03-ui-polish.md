# UI Polish Implementation Plan (Plan 7)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Take the Plan 6 frontend from "functionally complete" to "actually pleasant" — give the Tiptap editor a real toolbar, replace the `window.prompt()` context-menu hack with a real component, wire drag-and-drop for the doc tree (the server `POST /api/docs/:id/move` is already there), add a `⌘K` command palette, and gate destructive controls on the user's effective role so non-owners never see buttons that would 403.

**Architecture:**
- **Editor toolbar** — a new `<EditorToolbar editor={editor} />` next to the existing `<EditorContent>`. Buttons dispatch Tiptap commands. The Link button uses a small inline popover (no separate dialog component) to keep the surface minimal. Toolbar visibility is gated by `role !== "viewer"`.
- **Context menu** — a single `<ContextMenu>` primitive using a portal + click-outside + Escape-to-close. Tree rows wire `onContextMenu` to open it with a list of items relevant to the row's effective role.
- **Drag and drop** — `@dnd-kit/core` + `@dnd-kit/sortable`. The tree wraps rows in `useSortable`; drop position is mapped to `{ parent_id, before_id }` for `docsApi.move`. Reorder is optimistic (TanStack Query setQueryData) with a rollback on server error.
- **Command palette** — `⌘K` (or `Ctrl+K`) toggles a modal with a single text input. Actions: filter visible docs, create-doc, go-to-settings, go-to-members, log-out. Driven by Zustand UI state (slot already exists from Plan 6 T3).
- **Role gating** — a `useEffectiveRole(docId)` hook returns the doc's `effective_role` from the existing `["doc", id]` query. Workspace-level role comes from the session. Components do `if (role === "viewer") return null` for destructive controls.

**Tech Stack:** `@dnd-kit/core` 6 + `@dnd-kit/sortable` 8 (the only new deps). All other work uses existing React 18 + TanStack Query + Tiptap.

## Conventions

- **Component file = one default export.** Sub-components stay private unless explicitly exported. Tests for a component live next to it as `Foo.test.tsx`.
- **`data-testid`** — every new interactive control gets a stable testid. Use kebab-case: `toolbar-bold`, `cmdk-input`, `ctx-rename`.
- **Tiptap commands** — always use `editor.chain().focus().toggleX().run()` so the editor regains focus after a click. Otherwise the next keystroke is lost.
- **dnd-kit sensors** — use `PointerSensor` + `KeyboardSensor` so keyboard users can reorder too.
- **Modal a11y** — `role="dialog"`, focus trap via a small `useEffect`. Don't pull in a focus-trap library; the palette is small enough to do it manually.
- **Optimistic mutations** — for tree reorder, write into the `["docs"]` query cache immediately, then `mutate`. On error, invalidate to roll back.
- **Bundle hygiene** — after each task, eyeball `pnpm build` output. If main bundle pushes past 250 KB gzipped, find what got pulled in and trim.

```tsx
import { createPortal } from "react-dom";
import { useEffect, useRef } from "react";

export type ContextMenuItem = {
  label: string;
  testId: string;
  destructive?: boolean;
  disabled?: boolean;
  onSelect: () => void;
};

export function ContextMenu({
  x,
  y,
  items,
  onClose,
}: {
  x: number;
  y: number;
  items: ContextMenuItem[];
  onClose: () => void;
}) {
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    const onDoc = (e: MouseEvent) => {
      if (!ref.current?.contains(e.target as Node)) onClose();
    };
    const onKey = (e: KeyboardEvent) => {
      if (e.key === "Escape") onClose();
    };
    document.addEventListener("mousedown", onDoc);
    document.addEventListener("keydown", onKey);
    return () => {
      document.removeEventListener("mousedown", onDoc);
      document.removeEventListener("keydown", onKey);
    };
  }, [onClose]);

  return createPortal(
    <div
      ref={ref}
      role="menu"
      data-testid="context-menu"
      style={{
        position: "fixed",
        top: y,
        left: x,
        background: "white",
        border: "1px solid #e5e5e5",
        borderRadius: 4,
        boxShadow: "0 4px 12px rgba(0,0,0,0.1)",
        minWidth: 160,
        padding: 4,
        zIndex: 50,
      }}
    >
      {items.map((it) => (
        <button
          key={it.testId}
          type="button"
          role="menuitem"
          data-testid={it.testId}
          disabled={it.disabled}
          onClick={() => {
            onClose();
            it.onSelect();
          }}
          style={{
            display: "block",
            width: "100%",
            textAlign: "left",
            padding: "6px 10px",
            border: "none",
            background: "transparent",
            color: it.destructive ? "#b00020" : "inherit",
            cursor: it.disabled ? "not-allowed" : "pointer",
          }}
        >
          {it.label}
        </button>
      ))}
    </div>,
    document.body,
  );
}
```

Add to `tree.ts`:

```ts
import type { Doc } from "../../lib/validators";

/** Given a flat doc list, produce a flat list with one doc moved to a new
 *  parent (drop-onto-row), inserted at the end of that parent's children. */
export function reorderInto(docs: Doc[], movedId: string, newParentId: string | null): Doc[] {
  return docs.map((d) => (d.id === movedId ? { ...d, parent_id: newParentId } : d));
}

/** Given the dropTarget's neighbor info, return the move args expected by
 *  POST /api/docs/:id/move. Caller decides "before" vs "after" based on the
 *  drop position (top half vs bottom half of the row). */
export function moveArgs(
  target: Doc | null,
  position: "before" | "after" | "into",
): { parent_id?: string | null; before_id?: string; after_id?: string } {
  if (!target) return { parent_id: null };
  switch (position) {
    case "before": return { parent_id: target.parent_id, before_id: target.id };
    case "after":  return { parent_id: target.parent_id, after_id: target.id };
    case "into":   return { parent_id: target.id };
  }
}
```

const move = useMutation({

  onMutate: async (a) => {

type Action = { id: string; label: string; kind: "doc" | "nav" | "action"; run: () => void };

export function CommandPalette() {

  const close = useUi((s) => s.closePalette);

  useEffect(() => {
    const onKey = (e: KeyboardEvent) => {
      if ((e.metaKey || e.ctrlKey) && e.key.toLowerCase() === "k") {
        e.preventDefault();
        useUi.getState().paletteOpen ? close() : useUi.getState().openPalette();
      } else if (e.key === "Escape" && useUi.getState().paletteOpen) {
        close();
      }
    };
    document.addEventListener("keydown", onKey);
    return () => document.removeEventListener("keydown", onKey);

    const docList = docs.data && "ok" in docs.data ? docs.data.ok : [];

      label: d.title,

      data-testid="cmdk"

        paddingTop: "10vh", zIndex: 60,

          data-testid="cmdk-input"
          autoFocus
          value={q}
          onChange={(e) => setQ(e.target.value)}
          placeholder="Type to search…"
          onKeyDown={(e) => {
            if (e.key === "ArrowDown") { e.preventDefault(); setCursor((c) => Math.min(c + 1, filtered.length - 1)); }
            else if (e.key === "ArrowUp") { e.preventDefault(); setCursor((c) => Math.max(c - 1, 0)); }
            else if (e.key === "Enter") { filtered[cursor]?.run(); }
          }}
          style={{ width: "100%", padding: 10, border: "1px solid #e5e5e5", borderRadius: 4 }}
        />
        <ul data-testid="cmdk-list" style={{ listStyle: "none", margin: 0, padding: 0, marginTop: 8, maxHeight: 320, overflow: "auto" }}>
          {filtered.map((a, i) => (
            <li key={a.id}>
              <button
                type="button"
                data-testid={`cmdk-item-${a.id}`}
                onClick={a.run}

                  cursor: "pointer",
                }}
              >
                {a.label}
              </button>
            </li>
          ))}
          {filtered.length === 0 && (

      </div>

  );
}
